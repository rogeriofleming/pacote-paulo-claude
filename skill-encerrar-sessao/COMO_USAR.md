# Como instalar e usar o Encerrar Sessão

## O que é

Um protocolo de **fechamento de sessão de trabalho** — mas a ideia-chave é: encerrar **não
é faxina, é colheita**. Ele relê a conversa inteira e roteia cada aprendizado pro lugar
único e certo:

- correção sobre **como o agente deve agir** → memória (feedback)
- fato novo sobre você/pessoas/projeto → memória
- conhecimento técnico reutilizável → documento
- método novo que vale repetir → vira skill
- mudança de status → o doc de status (fonte única)

Depois atualiza o `WORKFLOW.md` (estado, não diário), escreve o `PROXIMA_SESSAO.md`,
arquiva um resumo se a sessão foi pesada, limpa temporários e sincroniza o git.

O princípio que sustenta tudo: **uma informação mora em UM só documento**. Duplicou →
dessincroniza → vira mentira na sessão seguinte.

Combina com o **Iniciar Sessão** (neste mesmo repo) — um abre, o outro fecha.

## Instalação

Copie a pasta pra `.claude/skills/encerrar-sessao/` na raiz do seu projeto:
```
.claude/skills/encerrar-sessao/SKILL.md
```

## Como acionar

"encerra a sessão", "fecha o dia", "vamos encerrar".

## Adapte à sua realidade

Os nomes de arquivo (`WORKFLOW.md`, `PROXIMA_SESSAO.md`, `sessoes/`) e o passo opcional de
rede de notas (Obsidian/wikilinks) são sugestões — troque pelos seus. O que não muda é a
espinha: **colher os aprendizados + manter fonte única + deixar o estado confiável pra
próxima sessão.**
